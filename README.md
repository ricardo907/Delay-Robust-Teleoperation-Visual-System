# Delay-Robust-Teleoperation-Visual-System
Delay-Robust Teleoperation System with Digital Twin Prediction, Risk-Adaptive Shared Control, and Visual Analytics

This project presents a complete simulation-based teleoperation system designed to improve the safety, stability, and robustness of robotic manipulation under communication delay and operational uncertainty. The system is developed as a Windows-compatible Python application with an integrated visual interface, allowing users to run interactive simulations, benchmark multiple control strategies, and automatically generate result visualizations and reports.

The core objective of the project is to address a common challenge in remote robot operation: when communication delay, input noise, and environmental constraints are present, direct teleoperation often becomes unstable, inaccurate, and prone to collision. To solve this problem, the system combines three key components. First, a digital twin prediction module estimates the delayed future state of the robot, reducing state mismatch caused by latency. Second, a risk-adaptive shared control module dynamically blends human command and autonomous assistance according to task risk, uncertainty, and input stability. Third, a safety filtering layer constrains unsafe motions and improves collision avoidance in cluttered or narrow environments.

The system supports multiple representative manipulation tasks, including obstacle-aware pick-and-place, narrow-passage transfer, and precision peg-in-hole style motion. It also includes several baseline methods, such as direct teleoperation, prediction-only control, and fixed-weight shared control, enabling systematic comparison against the proposed approach. In addition, the project provides an experimental pipeline for running benchmark suites under different delay settings and operator behaviors, with automatic export of trajectory plots, CSV metrics, summary figures, GIF-based visual demos, and HTML reports.

From a software perspective, the project is structured as a modular research platform rather than a simple demo. It contains dedicated modules for scenario generation, operator modeling, prediction, controller design, safety logic, simulation execution, visualization, experiment automation, and reporting. This makes the system suitable not only for demonstration purposes, but also for academic research, algorithm validation, and future integration with a higher-fidelity robotics simulator such as CoppeliaSim.

Overall, the project delivers a complete and extensible teleoperation research framework that combines simulation, control, visualization, and evaluation in one package. It is especially suitable for studying delay-aware robotic manipulation and for building publication-oriented experiments without requiring a Linux-based robotics platform.
